Auto <p>'s and <br>'s

Nesting shortcodes

#Case 1

 [cl-boxout title="Sample"] This is a cutout. [cl-button] [/cl-boxout]

Expected

<div class="cl-boxout"> <h1>Sample</h1> <p> This is a cutout. <a class="cl-button">Explore</a> </p> </div>

Actual

<div class="cl-boxout"> <h1>Sample</h1> <p> <br> This is a cutout. <br> <a class="cl-button">Explore</a> <br> </p> </div>

#Case 2

 [cl-boxout title="Sample"] This is a cutout. [cl-button] [/cl-boxout]

Expected

<div class="cl-boxout"> <h1>Sample</h1> <p> This is a cutout. <a class="cl-button">Explore</a> </p> </div>

Actual

<div class="cl-boxout"> <h1>Sample</h1> <p></p> <p>This is a cutout.</p> <p> <a class="cl-button">Explore</a> <br> </p> </div>

#Case 3

Putting a double line break between paragraphs is required to work as expected.

[cl-boxout title="Sample"]This is a cutout. A second paragraph.[cl-button][/cl-boxout]

Expected

<div class="cl-boxout"> <h1>Sample</h1> <p>This is a cutout.</p> <p> A second paragraph. <a class="cl-button">Explore</a> </p> </div>

Actual

<div class="cl-boxout"> <h1>Sample</h1> <p>This is a cutout.</p> <p> A second paragraph. <a class="cl-button">Explore</a> </p> </div>

Things get trickier when nesting complex components inside tiles.

#Case 4

 [cl-tiles] [cl-boxout title="Sample"]This is a cutout.[cl-button][/cl-boxout] [cl-card title="Sample" body="Here's a sample card."] [/cl-tiles]

Expected

<div class="cl-tiles thirds"> <div class="cl-boxout"> <h1>Sample</h1> <p> This is a cutout. <a class="cl-button">Explore</a> </p> </div> <a class="cl-card"> <h1>Sample</h1> <p>Here's a sample card.</p> <span class="cl-button">Explore</span> </a> </div>

Actual

<div class="cl-tiles thirds"> <br> <div class="cl-boxout"> <h1>Sample</h1> <p> This is a cutout. <a class="cl-button">Explore</a> </p> </div> <br> <a class="cl-card"> <h1>Sample</h1> <p>Here's a sample card.</p> <span class="cl-button">Explore</span> </a> <br> </div>

#Case 5

Writting everything on one line works as expected.

[cl-tiles][cl-boxout title="Sample"]This is a cutout.[cl-button][/cl-boxout][cl-card title="Sample" body="Here's a sample card."][/cl-tiles]

Expected

<div class="cl-tiles thirds"> <div class="cl-boxout"> <h1>Sample</h1> <p> This is a cutout. <a class="cl-button">Explore</a> </p> </div> <a class="cl-card"> <h1>Sample</h1> <p>Here's a sample card.</p> <span class="cl-button">Explore</span> </a> </div>

Actual

<div class="cl-tiles thirds"> <div class="cl-boxout"> <h1>Sample</h1> <p> This is a cutout. <a class="cl-button">Explore</a> </p> </div> <a class="cl-card"> <h1>Sample</h1> <p>Here's a sample card.</p> <span class="cl-button">Explore</span> </a> </div>

#Case 6

When including a cutout with multiple paragraphs, a double line break is needed between the paragraphs in order to work as expected.

[cl-tiles][cl-boxout title="Sample"]This is a cutout. A second paragraph.[cl-button][/cl-boxout][cl-card title="Sample" body="Here's a sample card. It's pretty simple."][/cl-tiles]

Expected

<div class="cl-tiles thirds"> <div class="cl-boxout"> <h1>Sample</h1> <p>This is a cutout.</p> <p> A second paragraph. <a class="cl-button">Explore</a> </p> </div> <a class="cl-card"> <h1>Sample</h1> <p>Here's a sample card.</p> <span class="cl-button">Explore</span> </a> </div>

Actual

<div class="cl-tiles thirds"> <div class="cl-boxout"> <h1>Sample</h1> <p>This is a cutout.</p> <p> A second paragraph. <a class="cl-button">Explore</a> </p> </div> <a class="cl-card"> <h1>Sample</h1> <p>Here's a sample card.</p> <span class="cl-button">Explore</span> </a> </div>

Anchors with inline content

Cards

#Case 1

When hard-coding cards, WordPress will wrap the header and paragraph in one card, the button in another, and add an empty card before the whole thing.

<a href="#" class="cl-card"> <h1>Card</h1> <p>Some card text</p> <span class="cl-button">Button</span> </a>

Expected

<a href="#" class="cl-card"> <h1>Card</h1> <p>Some card text</p> <span class="cl-button">Button</span> </a>

Actual

<p> <a href="#" class="cl-card"></a> </p> <a href="#" class="cl-card"> <h1>Card</h1> <p>Some card text</p> </a> <p> <a href="#" class="cl-card"> <span class="cl-button">Button</span> <br> </a> </p>

#Case 2

Writing the card HTML on one line produces even stranger results.

<a href="#" class="cl-card"><h1>Card</h1><p>Some card text</p><span class="cl-button">Button</span></a>

Expected

<a href="#" class="cl-card"> <h1>Card</h1> <p>Some card text</p> <span class="cl-button">Button</span> </a>

Actual

<p> <a href="#" class="cl-card"> <br> </a> </p> <h1> <a href="#" class="cl-card">Card</a> </h1> <a href="#" class="cl-card"> <p>Some card text</p> </a> <p> <a href="#" class="cl-card"> <span class="cl-button">Button</span> </a> </p>
