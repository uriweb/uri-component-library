Buttons

Buttons focus user attention on and allow them to take major actions.

  • Usage
  • Appearance

Usage

Buttons are implemented with the class cl-button

<a class="cl-button" href="#">Explore</a>
Explore

To give a button emphasis over others, use the class prominent

<a class="cl-button prominent" href="#">Explore</a>
Explore

Shortcode Support

Buttons can be implemented with the shortcode cl-button

[cl-button link="sample.html"]

Attributes

link (string) (required)

The URL

text (string) (optional)

The text for the button
(default value: 'Explore')

tooltip (string) (optional)

Set a tooltip
(default value: none)

prominent (bool) (optional)

Set to give the button prominence
(default value: false)

class (string) (optional)

Set custom CSS class(es)
(default value: none)

WYSIWYG Editor Support

With the WYSIWYG plugin installed, buttons can be added using the WYSIWYG editor.

The icon to add a button appears in the WYSIWYG toolbar. Clicking this icon opens a dialogue where different button attributes can be easily set.

The completed button will appear fully styled in the editor window. To edit it, simply double-click the button to reopen the editor dialogue.

Appearance

A button's appearance depends on the layout framework or component it's in.

Note that buttons in cards and heros are never prominent.
