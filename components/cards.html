Cards

A card is an elevated, clickable container with abridged contents and a single action, serving as an entry-point to more detailed information.

Cards do not contain any links. Clicking the card is the only action a user can take.

Action Card

Action cards are vertical cards that prompt user action with an optional image, summary language, and a button.

Action cards are implemented with the class cl-card

<a class="cl-card" href=""> <h1>Title</h1> <p>Paragraph text.</p> <span class="cl-button">Button</span> </a>

Shortcode Support

Action cards can be implemented with the shortcode cl-card

[cl-card title="A Great Title" body="The body text" link="sample.html"]

WYSIWYG Editor Support

With the WYSIWYG plugin installed, cards can be added using the WYSIWYG editor.

The icon to add a card appears in the WYSIWYG toolbar. Clicking this icon opens a dialogue where different card attributes can be easily set.

The completed card will appear fully styled in the editor window. To edit it, simply double-click the card to reopen the editor dialogue.

Usage

Card content is isolated.

The purpose of cards is to highlight information with the goal of enticing user action. Cards should not be used simply to emphasize some content or act as a header.

A good test is this: If removing the card from the page makes it more difficult to understand other page content, the card is not being used properly.

Don't

Containing primary information in a card degrades its relationship with secondary information below. Removing the card makes the subject of other column content unclear.

Instead

In this case, use native tile features. There are no nested elements, and the subject of all information is clear.

Action cards and detail cards can live inside tiles.

To place a card in a tile, use the tile framework.

<div class="cl-tiles thirds"> <div> ... <a class="cl-card">...</a> ... </div> ... </div>

Construction

Cards are elevated.

Cards exist above the page, not on it. A box shadow is used to achieve this appearance.

Do

Use a box shadow to elevate the card.

Don't

This is not a card. There is no indication that the content exists above the page.

Don't

Do not place a border around the content to emulate a card. This content still exists on the page plane.

Card elevation changes when hovering over and clicking them.

Increasing elevation upon hover and clicking provides the user with feedback for their actions and improves user experience. Elevation changes are achieved with a box shadow and position shift.

Resting

  • Box shadow
  • x-offset: none
  • y-offset: 0.1rem
  • Blur radius: 0.3rem
  • Color: $black #000, 15% alpha
  • Position
  • Top: 0px
