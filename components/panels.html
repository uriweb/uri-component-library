Panels

Panels are large content containers that live on the page plane. They are constructed of two panes: one displaying text and the other displaying an image. The panes sit side-by-side when space allows, and stack at narrow browser widths. Unlike cards, there is no single user action for a panel and the panel itself is not clickable; text can contain multiple links and user actions.

  • Design

Design

Panels require an image and are top level components, so they cannot live in tiles.

Panels are implemented with the class cl-panel

<div class="cl-panel"> <figure> <img> </figure> <article> <h1>Title</h1> <p>Paragraph text.</p> <a class="cl-button" href="">Button</a> ... </article> </div>

Note that the image must be nested in <figure> tags. In the full-width layout, the height of the figure is dependent on the height of the article, and image covers the figure. At narrow widths, the figure is stacked on top of the article, and the image is not cropped.

Panel panes can be swapped

By default, the left pane displays text and the right pane displays an image. The layout can be reversed by adding the class reverse to the panel. This will not change how the panes behave at narrow browser widths.

Note that the figure block must always come before the article block, regardless of layout.

<div class="cl-panel reverse"> <figure> <img> </figure> <article> ... </article> </div>

Shortcode Support

Panels can be implemented with the shortcode cl-panel

[cl-panel title="A Great Panel" img="sampleimage.jpg"]Some body text.[/cl-panel]

The panel shortcode wraps the content in <p> tags. To include multiple paragraphs, place </p><p> between each one.

[cl-panel title="A Great Panel" img="sampleimage.jpg"]This is the first paragraph.</p><p>This is the second paragraph.</p><p>Why not a third?[/cl-panel]

Attributes

img (string) (required)

The source URL for the image

alt (string) (optional)

Set alternate text for the image
(default value: none)

title (string) (optional)

The text for a title
(default value: none)

reverse (bool) (optional)

Set to reverse the panel layout
(default value: false)

class (string) (optional)

Set custom CSS class(es)
(default value: none)

WYSIWYG Editor Support

With the WYSIWYG plugin installed, panels can be added using the WYSIWYG editor.

The icon to add a panel appears in the WYSIWYG toolbar. Clicking this icon opens a dialogue where different panel attributes can be easily set.

The completed panel will appear fully styled in the editor window. To edit it, simply double-click the panel to reopen the editor dialogue.
